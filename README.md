# Ubuntu Hardening Project

**Project Goal:** Secure a fresh Ubuntu installation (bare metal or VM) by applying standard security best practices. All steps have been tested and verified on a clean system.

---

## Table of Contents
1. Update and Upgrade the System
2. Create a Non-Root User
3. Enable Firewall (UFW)
4. Secure SSH
5. Enable Automatic Security Updates
6. Install Security Tools
7. Disk Encryption (Optional)
8. Disable Unnecessary Services
9. Browser & Application Security
10. Enable Two-Factor Authentication (Google Authenticator)

---

## Why This Project
This repository provides a clear guide to hardening a fresh Ubuntu installation. It helps reduce the attack surface, improve login security, and maintain a safer system for day-to-day use.

---

## Notes
- Always back up important data before making major system changes.
- Avoid pushing sensitive files (like private keys or passwords) to GitHub.
- This repository can serve as a template for future Ubuntu installations.
